title: Quickstart

This guide walks you through cloning the MidnightBSD source tree, building the userland and kernel, installing everything, and preparing for first boot. It follows the standard upgrade procedure documented at the top of the source `Makefile`.

<Note>
  You must be running MidnightBSD (or a compatible BSD) to build natively. Cross-compilation from Linux is possible but requires additional toolchain setup — see the [Cross-Compilation](/building/cross-compilation) guide.
</Note>

## Prerequisites

### Disk space

| Location | Minimum free space | Notes |
|---|---|---|
| `/usr/obj` | 6 GB | Object files for a single-arch world+kernel build |
| `/usr/src` | 2 GB | Full source checkout |
| `/usr` | 1 GB additional | For `installworld` output when `DESTDIR` is not set |

A complete `universe` build (all architectures) required 167 GB as of late 2018; ZFS lz4 compression reduces that to roughly 81 GB.

### Tools required

- `git` — to clone the repository
- `bmake` — MidnightBSD's build make (already present on a running system)
- A working C compiler (`clang` is the default; the build will bootstrap one if needed)
- Sufficient swap space if RAM is limited (a world build is memory-intensive)

## Build steps

<Steps>
  <Step title="Clone the source tree">
    Stable branches are recommended over `master` for production systems. Replace `3.2` with the desired stable branch:

    ```bash
    cd /usr
    git clone https://github.com/MidnightBSD/src.git -b stable/3.2
    ```

    To switch branches later:

    ```bash
    cd /usr/src
    git checkout master
    ```
  </Step>

  <Step title="Build the world (userland)">
    `buildworld` rebuilds everything in the MidnightBSD system from source **except** the kernel, kernel modules, and `/etc`. Object files are written to `/usr/obj`.

    ```bash
    cd /usr/src
    make buildworld
    ```

    To speed up the build on a multi-core machine, pass `-j` with the number of parallel jobs:

    ```bash
    make -j$(sysctl -n hw.ncpu) buildworld
    ```

    <Tip>
      Set `MAKEOBJDIRPREFIX` in your environment (not in `make.conf`) to redirect object files to an alternate path, for example `export MAKEOBJDIRPREFIX=/fast-disk/obj`.
    </Tip>
  </Step>

  <Step title="Build the kernel">
    `buildkernel` rebuilds the kernel and all loadable modules. The default configuration is `GENERIC`.

    ```bash
    make buildkernel KERNCONF=GENERIC
    ```

    Replace `GENERIC` with the name of your custom configuration file if you have one. Configuration files live in `sys/<arch>/conf/`.

    ```bash
    # Example: custom config at sys/amd64/conf/MYKERNEL
    make buildkernel KERNCONF=MYKERNEL
    ```
  </Step>

  <Step title="Install the new kernel">
    Install the freshly built kernel before rebooting into single-user mode for `installworld`:

    ```bash
    make installkernel KERNCONF=GENERIC
    ```

    <Warning>
      Installing a new kernel while running the old userland is safe. Installing a new userland (`installworld`) while running the old kernel is **not** recommended — always install the kernel first, then reboot.
    </Warning>
  </Step>

  <Step title="Reboot into single-user mode">
    Reboot the system. At the loader prompt, boot into single-user mode:

    ```
    boot -s
    ```

    Single-user mode prevents services from interfering with the userland upgrade.
  </Step>

  <Step title="Run mergemaster pre-pass">
    Before installing the new world, run `mergemaster -p` to update critical `/etc` files that `installworld` depends on:

    ```bash
    cd /usr/src
    mergemaster -p
    ```
  </Step>

  <Step title="Install the world (userland)">
    Install everything built by `buildworld` into the live system:

    ```bash
    cd /usr/src
    make installworld
    ```

    <Note>
      To install to an alternate root (e.g. a staging directory or a different disk), set `DESTDIR`:
      ```bash
      make installworld DESTDIR=/mnt/newroot
      ```
    </Note>
  </Step>

  <Step title="Run mergemaster (full pass)">
    Merge configuration file changes from the new sources into `/etc`:

    ```bash
    mergemaster
    ```

    Common flags: `-i` auto-installs files with no local changes, `-U` auto-upgrades files the build system owns, `-F` keeps forward-compatible changes.

    ```bash
    mergemaster -i -U
    ```
  </Step>

  <Step title="Remove obsolete files and directories">
    After installing the new world, check for and remove files that are no longer part of the system:

    ```bash
    make check-old        # preview what will be removed
    make delete-old       # remove obsolete files and directories
    ```
  </Step>

  <Step title="Final reboot">
    Reboot into the fully upgraded system:

    ```bash
    reboot
    ```

    Once running on the new kernel and userland, clean up obsolete shared libraries that may still be referenced by third-party software:

    ```bash
    cd /usr/src
    make delete-old-libs
    ```
  </Step>
</Steps>

## Quick reference: key make targets

| Target | Description |
|---|---|
| `buildworld` | Build entire userland from source |
| `installworld` | Install the built userland |
| `buildkernel` | Build kernel and modules |
| `installkernel` | Install kernel and modules |
| `world` | `buildworld` + `installworld` (requires `HISTORICAL_MAKE_WORLD` or `DESTDIR`) |
| `kernel` | `buildkernel` + `installkernel` |
| `check-old` | List obsolete files/directories/libraries |
| `delete-old` | Delete obsolete files and directories |
| `delete-old-libs` | Delete obsolete shared libraries |

For full details on all available targets, see the [build(7)](https://www.midnightbsd.org/man/man7/build.7.html) manual page or read the comments at the top of `/usr/src/Makefile`.
