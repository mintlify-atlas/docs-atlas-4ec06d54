## What is MidnightBSD?

MidnightBSD is a Unix-like open source operating system derived from FreeBSD. It includes a complete kernel, a full userland with hundreds of utilities, comprehensive system libraries, and a build infrastructure targeting multiple hardware architectures. MidnightBSD aims to be a stable, security-focused operating system suitable for desktop, server, and embedded use.

The source tree contains everything needed to build a complete MidnightBSD system from scratch — from the kernel and device drivers to the shell, compiler toolchain, cryptographic libraries, and package manager.

## Source tree structure

The MidnightBSD source tree is organized into the following top-level directories:

| Directory | Contents |
|-----------|----------|
| `bin/` | Core user commands (ls, cp, sh, ps, etc.) |
| `sbin/` | System administration commands (ifconfig, mount, fsck, etc.) |
| `usr.bin/` | General user utilities (grep, awk, make, tar, etc.) |
| `usr.sbin/` | System administration utilities (sshd, cron, etc.) |
| `lib/` | System libraries (libc, libpthread, libssl, etc.) |
| `sys/` | Kernel source code |
| `etc/` | Configuration file templates |
| `share/` | Shared data, man pages, locales |
| `contrib/` | Third-party contributed software |
| `crypto/` | Cryptographic libraries and commands |
| `kerberos5/` | Kerberos 5 (Heimdal) package |

## Getting the source

```bash
# Clone the stable branch (recommended for production use)
git clone https://github.com/midnightbsd/src.git -b stable/3.2 /usr/src

# Or clone master for development
git clone https://github.com/midnightbsd/src.git /usr/src
```

<Note>
Most users should check out a stable branch rather than `master`. The `master` branch is the active development branch and may contain incomplete or experimental changes.
</Note>
